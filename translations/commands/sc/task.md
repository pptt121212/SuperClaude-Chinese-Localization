# /sc:task - 通过智能工作流管理和委派执行复杂任务

## 触发器
- 需要多代理协调和委派的复杂任务
- 需要结构化工作流程管理和跨会话持久性的项目
- 需要智能MCP服务器路由和领域专业知识的操作
- 受益于系统执行和渐进增强的任务

## 用法
```
/sc:task [操作] [目标] [--strategy systematic|agile|enterprise] [--parallel] [--delegate]
```

## 行为流程
1. **分析**: 解析任务要求并确定最佳执行策略
2. **委派**: 路由到适当的MCP服务器并激活相关角色
3. **协调**: 通过智能工作流程管理和并行处理执行任务
4. **验证**: 应用质量门禁和全面任务完成验证
5. **优化**: 分析性能并提供增强建议

关键行为：
- 跨架构、前端、后端、安全、devops领域的多角色协调
- 智能MCP服务器路由（Sequential、Context7、Magic、Playwright、Morphllm、Serena）
- 具有渐进任务增强和跨会话持久性的系统执行
- 具有层次化分解和依赖管理的高级任务委派

## MCP集成
- **Sequential MCP**: 复杂多步任务分析和系统执行规划
- **Context7 MCP**: 框架特定模式和实施最佳实践
- **Magic MCP**: UI/UX任务协调和设计系统集成
- **Playwright MCP**: 测试工作流程集成和验证自动化
- **Morphllm MCP**: 大规模任务转换和基于模式的优化
- **Serena MCP**: 跨会话任务持久性和项目内存管理

## 工具协调
- **TodoWrite**: 跨Epic → Story → Task级别的层次化任务分解和进度跟踪
- **任务**: 复杂多代理协调和子任务管理的高级委托
- **读/写/编辑**: 任务文档和实施协调
- **sequentialthinking**: 复杂任务依赖分析的结构化推理

## 关键模式
- **任务层次**: Epic级别目标 → Story协调 → 任务执行 → Subtask粒度
- **策略选择**: 系统化（全面）→ 敏捷（迭代）→ 企业（治理）
- **多代理协调**: 角色激活 → MCP路由 → 并行执行 → 结果集成
- **跨会话管理**: 任务持久性 → 上下文连续性 → 渐进增强

## 示例

### 复杂功能开发
```
/sc:task create "企业认证系统" --strategy systematic --parallel
# 具有多领域协调的综合任务分解
# 激活架构师、安全、后端、前端角色
```

### 敏捷冲刺协调
```
/sc:task execute "功能积压" --strategy agile --delegate
# 具有智能委派的迭代任务执行
# 冲刺连续性的跨会话持久性
```

### 多领域集成
```
/sc:task execute "微服务平台" --strategy enterprise --parallel
# 具有合规验证的企业级协调
- 跨多个技术领域的并行执行
```

## 边界

**将会：**
- 通过多代理协调和智能委派执行复杂任务
- 提供具有跨会话持久性的层次化任务分解
- 协调多个MCP服务器和角色以获得最佳任务结果

**将不会：**
- 执行不需要高级编排的简单任务
- 为速度或便利性而妥协质量标准
- 在没有适当验证和质量门禁的情况下操作
